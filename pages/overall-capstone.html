Project Inspiration

I decided to create a web-based collaborative diagram editor because it mixed three of my favorite interests in computer science: web development, networking, and UI/UX. Creating a website requires web development, and the user interface and experience are core components of any website. The collaboration feature allows for the integration of networking.

I chose to make a diagram editor because it is a simple concept with a high amount of technical potential. I didn’t want to get sidetracked by an ambitious concept because I would have been more likely to get distracted with the concept itself rather than the technical component. A diagram editor is a beneficial technical challenge due to having to develop a front-end and back-end. A front-end is important for the graphic and visual demands of diagrams, and a back-end is required for networking and collaboration.

Using the Double Diamond Methodology

I went into the project with the intention of using the double diamond design methodology. My goal was to follow a structured methodology to gain insight into what project management is like at an industry-level. Something I failed to consider was that the goal of a professional design methodology and the goal of the capstone are different. The goal of the former is to create a useful and viable product, while the capstone’s goal is to learn about development. The dissonance between goals created issues early on. I tried to steadily create output even when the output wasn’t useful to the project. After realizing that I wasn’t making much progress, I took a more balanced approach. I am now learning and practicing to balance creating output that is useful and learning what is necessary to create that output.

Another factor that contributed to the allure of the double diamond methodology was my desire for structure. Having structure gives me comfort and allows me to focus solely on the project itself. However, perfect structure is an unrealistic expectation and something I won’t encounter in the professional world. So, part of dropping the double diamond methodology was to force myself to become comfortable having to navigate an unfamiliar project space. It is not preferable, but becoming adaptable to real world situations is important to me and something I am improving at.

Next Steps

After a couple of meetings with a professional software developer, I learned more about the methodologies and technologies necessary for optimized performance. I plan to research more in-depth about React and Canvas API. React is the core technology of the front-end, so understanding its technical implementation allows me to optimize the user experience and DOM tree. Canvas API is the main handler of 2D graphics, which is responsible for all the visuals of diagramming. Becoming adept at React and Canvas API gives me useful front-end knowledge, useful for any web developer. Implementing these technologies helps ElysianMap’s goal of performance.
